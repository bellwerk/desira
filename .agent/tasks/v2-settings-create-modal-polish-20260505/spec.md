# V2 Settings and Create Modal Polish

## Acceptance Criteria
- AC1: Settings and account logout buttons use the restrained primary-button scale while preserving their destructive sign-out meaning.
- AC2: Settings/account modal panels do not leave excessive empty space below the logout action.
- AC3: The topbar user pill no longer reserves excessive leading space before the display name.
- AC4: The create-list modal no longer forces a large empty bottom area after the submit button.
- AC5: Create-list quick-start chips show only user-facing labels, with no confusing letter abbreviations.
- AC6: The create-list event date control uses an app-styled dark calendar surface instead of the native date picker popup.
- AC7: Existing create-list behavior and settings sign-out behavior remain unchanged.
- AC8: Lint, typecheck, build, unit tests, and browser smoke checks are recorded.
